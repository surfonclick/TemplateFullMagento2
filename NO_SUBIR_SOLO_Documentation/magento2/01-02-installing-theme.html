2. Installing The Package Theme

Back to top

If you already had a Magento store and want to apply the theme, follow the instruction below. To install the theme you only need to upload all the theme files on your server and then enable the theme in the admin panel. But to avoid unexpected problems after installation, your Magento should be prepared first.

How To Install Magento 2 and EM Everything on EC2 Amazon Server

  • 2.1. Prepare Your Magento
  • 2.2. Installing Theme after Magento is prepared

2.1. Prepare Your Magento

Step 1: Disable all cache in your Magento

IMPORTANT: When developing your Magento store, in order to be able to see changes immediately, you should completely disable the cache. You can enable it after you finish configuring your store
  1. Magento Cache. Please read more details in : this link
  2. Full Page cache and caching modules for Magento. Please read more details in : this link
  3. Any additional cache on your server, PHP cache engines, APC, etc Please read more details in : this link
  4. Disable Merge JavaScript Files and CSS Files in backend Stores > Configuration > Developer > JavaScript Settings, CSS Settings Please read more details in : this link

Step 2: Before you install, make sure your store is disabled (no page can be refreshed during the installation process).

Magento's root directory is place you can find basic files of Magento, such as: index.php and .htacess.
IMPORTANT: Creating backup before installing any themes or extensions in Magento is extremely important, especially if you are working on a live store. Backup Magento files and the store database

2.2. Installing Theme after Magento is prepared

This procedure doesn’t change core files of Magento

Step 1: Extracting file theme package on your computer and copy all files to your Magento folder. The theme folder after extracting, it should contain folders such as: app, pub.

Step 2: Using your FTP client upload all folders and files to your Magento's root directory Magento already contains the same directories - do not remove those directories. (You can upload your files to server by using Filezilla.) Directories extracted from theme package have to be merged with existing directories of Magento. If your system asks you whether you want to merge directories, you can confirm that you want to merge those directories.

Step 3: Set correct file permissions for folders as below image.

Step 4: Running the Setup Wizard , it is a multi-page wizard that enables you to go back and forward one page at a time.

Getting started

To install the Magento software using the Setup Wizard:

  1. Start a web browser.
  2. Enter the following URL in the browser’s address or location bar: http://<Magento host or IP>/<path to Magento root>/themesetup

    For example, if the Magento server’s IP address is 192.0.2.10 and you installed Magento 2 in the magento2 directory relative to the web server’s docroot, and you did not configure a Virtual Host, enter: http://192.0.2.10/magento2/setup On the initial page, click Agree and Set Up Magento.

Only use this when you can not install theme by Setup Wizard

  1. Using SSH to go the root folder (ex: var/www/html)
  2. Run the command "php -f bin/magento setup:upgrade"
  3. After that you, need set permission 777 for var/ and pub/ folder
    Using this command sudo chmod -R 777 var/ pub/
